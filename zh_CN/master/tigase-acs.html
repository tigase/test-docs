Tigase高级集群策略（ACS）–3.2.0版¶

1.设计和实施¶

1.1.ACS¶

ACS是我们通用的、商业化的集群策略，主要为典型的XMPP安装而设计，可以轻松地扩展到数百万甚至更多的在线用户，对集群节点没有限制。我们对代码进行的负载测试包括一个有100百万个账户的用户数据库和一个高达150人的平均名册，但这不是极限。

1.2.设计¶

集群策略是基于集群节点之间共享有关在线用户的信息。谁是在线的，用户在哪里连接。集群节点之间的通信是以最高优先级处理的，以确保在线用户数据的最小延迟。一个有效的同步机制允许集群节点之间的流量最小，并分发关于连接和断开用户的准确数据。

2.Tigase ACS SM安装¶

Tigase ACS SM组件默认与Tigase XMPP服务器版本（@-dist-max@风味的存档）一起提供，所以在配置中启用它就可以了。它也可以从tigase-acs发行包中获得。

下载存档后，只需将其解压，并将解压后的存档的jars/目录的内容复制到tigase-server/安装目录下的jars/目录，例如，在*nix系统下（假设存档被下载到Tigase服务器主目录）。

tar -xf tigase-acs-${version}.tar.gz
cp -R tigase-acs-${version}/jars/ tigase-server/jars/

3.Tigase ACS SM配置¶

为了使用高级集群策略，首先需要打开集群模式。

'cluster-mode' = true

然后需要启用一个ACS策略。

'sess-man' {
 strategy (class: tigase.server.cluster.strategy.OnlineUsersCachingStrategy) {}
 }

4.支持的组件¶

4.1.Tigase多用户聊天高级集群策略（ACS-MUC）。¶

4.1.1.概述¶

ACS for MUC允许MUC机房在Tigase XMPP服务器集群安装中的无缝集群。ACS for MUC是集群式MUC部署的必要条件。它提供了各种策略来处理流量和房间在集群中的分布，从而可以根据个人需求对部署进行微调。

4.1.2.Tigase ACS MUC配置¶

为了对MUC使用ACS，需要主要的高级集群策略。一旦它被启用，MUC组件的集群版本将在启动时被默认选择，因此不需要明确配置它（确保没有配置类）。

muc () {}

也可以通过以下配置明确配置类：

muc (class: tigase.muc.cluster.MUCComponentClustered) {}

在上述配置下，将使用默认的MUC集群策略。为了选择不同的策略，你必须在MUC组件豆中的策略豆中配置它/她的类。

muc () {
  strategy (class: tigase.muc.cluster.ShardingStrategy) {}
}
